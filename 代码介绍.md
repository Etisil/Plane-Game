# 代码框架结构说明

本文档说明当前飞机大战项目的主要代码结构。

## 文件结构

- `plane_main.py`
  - 游戏主流程文件。
  - 负责窗口、主循环、事件处理、碰撞检测、精灵更新、UI 绘制、暂停、难度选择、Boss WARNING、通关/失败流程。

- `plane_sprites.py`
  - 游戏对象定义文件。
  - 包含背景、英雄、普通敌机、Boss、子弹、Buff、血条、失败界面、通关界面等类。

- `danmaku_patterns.py`
  - 弹幕系统文件。
  - 包含 Boss 弹幕子弹、激光、延迟自机狙子弹、各种弹幕生成函数。

- `images/`
  - 图片资源目录。

- `music/`
  - 音效和背景音乐目录。

- `STCAIYUN.TTF`
  - 项目使用的字体文件，主要用于分数和部分界面。

## 主运行流程

程序入口在 `plane_main.py`：

```python
if __name__ == '__main__':
    PlaneGame.__start__()
```

启动流程：

1. 调用 `PlaneGame.__select_difficulty()` 显示难度选择界面。
2. 玩家选择 `normal`、`hard` 或 `lunatic`。
3. 创建 `PlaneGame(difficulty)`。
4. 调用 `start_game()` 进入主循环。

主循环中每帧执行：

1. `clock.tick(FRAME_PER_SEC)` 控制帧率。
2. `__event_handler()` 处理输入和定时器事件。
3. 根据状态进入：
   - 通关界面
   - 失败界面
   - 暂停界面
   - 正常游戏更新
4. `pygame.display.update()` 刷新屏幕。

## PlaneGame 主要职责

`PlaneGame` 是游戏控制中心。

主要成员：

- `self.screen`：pygame 显示窗口。
- `self.clock`：帧率控制。
- `self.enemy_group`：敌机组，普通敌机和 Boss 都在这里。
- `self.hero_group`：英雄组。
- `self.enemy_bullet_group`：旧式敌方子弹组。
- `self.danmaku_group`：弹幕和激光组。
- `self.buff1_group`：补给组。
- `self.bars`：血条列表。
- `self.boss_mode`：是否处于 Boss 战。
- `self.boss_warning`：是否处于 Boss 出现前 WARNING。
- `self.game_over`：是否失败。
- `self.game_clear`：是否通关。
- `self.paused`：是否暂停。

核心方法：

- `__event_handler()`
  - 处理退出、暂停、移动、射击、敌机生成、敌机发弹、Buff 生成。

- `__update_boss_warning()`
  - 根据分数判断是否进入 Boss WARNING。

- `__spawn_boss()`
  - 清场后创建 Boss。

- `__check_collide()`
  - 处理英雄子弹打敌机、敌机撞英雄、敌弹击中英雄、Buff 获取。

- `__update_sprites()`
  - 按绘制顺序更新背景、敌人、英雄、子弹、弹幕、UI。

- `__limit_danmaku_bullets()`
  - 控制弹幕数量，避免卡顿。

## 精灵对象结构

大多数游戏对象继承自 `pygame.sprite.Sprite`。

### GameSprite

`GameSprite` 是基础精灵类。

提供：

- 图片加载。
- `rect` 碰撞框。
- `speedx`、`speedy`。
- 基础 `update()` 移动逻辑。
- 默认血条。

### Background

背景滚动类。

两个背景对象上下拼接，向下滚动，离屏后回到屏幕上方。

### Hero

玩家飞机类。

负责：

- 移动和边界限制。
- 子弹等级。
- 子弹伤害。
- 发射子弹。
- 受伤闪白。
- 无敌帧。
- 低速模式中心判定点。
- 英雄爆炸动画。

英雄移动由 `plane_main.py` 读取键盘后设置：

```python
self.hero.speedx = x
self.hero.speedy = y
```

### Enemy

普通敌机类。

支持：

- 小型敌机。
- 中型敌机。
- 普通弹幕模式。
- 血条。
- 爆炸动画。

普通敌机弹幕入口是：

```python
Enemy.fire()
Enemy.fire_danmaku()
```

### Boss

Boss 类在 `plane_sprites.py` 中。

Boss 使用 5 阶段设计：

- `self.phase = 0`：第 1 阶段。
- `self.phase = 1`：第 2 阶段。
- `self.phase = 2`：第 3 阶段。
- `self.phase = 3`：第 4 阶段。
- `self.phase = 4`：第 5 阶段。

Boss 状态：

- `active`：正式战斗，可以受伤和发弹。
- `breaking`：阶段被击破，播放闪烁和音效。
- `filling`：新阶段血条从 0 填满。
- `defeating`：最终击破动画。

Boss 主要方法：

- `take_damage()`
  - 扣血并判断是否转阶段。

- `switch_phase()`
  - 切换到下一阶段，重置血条。

- `fire_danmaku()`
  - 根据当前阶段生成弹幕。

- `draw_phase_bar()`
  - 绘制环形阶段血条。

## 弹幕系统

弹幕相关代码已拆分到 `danmaku_patterns.py`。

### DanmakuBullet

Boss 弹幕子弹类。

特点：

- 使用浮点坐标移动。
- 统一弹幕尺寸。
- 支持不同显示样式：
  - 实心。
  - 空心。
  - 旋转椭圆。

### LaserBeam

激光类。

特点：

- 支持任意角度。
- 有预警阶段和伤害阶段。
- 使用线段距离进行碰撞判断。

### DelayedAimedBullet

延迟自机狙子弹。

流程：

1. 先停留在环形阵列中。
2. 等待一段时间。
3. 朝生成时锁定的玩家位置直线发射。

### DanmakuPattern

弹幕生成器，所有方法都是静态方法。

常见弹幕：

- `circle()`：圆形弹幕。
- `fan()`：扇形弹幕。
- `aimed()`：自机狙。
- `ordered_aimed_ring()`：环形自机狙。
- `rotating_stargate()`：旋转星门。
- `shrinking_trap_ring()`：扩散环。
- `waterwheel()`：水轮。
- `rotating_multi_directional()`：多方向旋转发射。

这些函数只负责生成子弹对象，真正加入精灵组是在 `PlaneGame.__add_enemy_projectile()` 中完成。

## 碰撞逻辑

碰撞集中在 `PlaneGame.__check_collide()`。

主要碰撞：

- 英雄子弹打敌机。
- 英雄撞敌机。
- 敌方弹幕打英雄。
- 敌方普通子弹打英雄。
- 英雄吃 Buff。

英雄判定框由：

```python
__hero_bullet_hitbox()
```

低速模式下，判定框会变成中心小白点。

激光碰撞特殊处理：

```python
__bullet_hits_hero()
```

如果敌弹有 `collides_with()`，就调用它；否则使用矩形碰撞。

## Boss 转阶段流程

Boss 某阶段血条归零后：

1. `Boss.take_damage()` 调用 `__start_phase_break()`。
2. Boss 进入 `breaking` 状态。
3. 播放击破音效并闪烁。
4. 达到 `BOSS_PHASE_BREAK_FRAMES` 后调用 `switch_phase()`。
5. 主循环通过 `__consume_boss_stage_notice()`：
   - 清空现场弹幕。
   - 显示 `STAGE X`。
   - 启动中心扩散环形波。
6. Boss 进入 `filling` 状态。
7. 血条从 0 填满。
8. 进入下一阶段 `active`。

## 难度系统

难度选择在 `plane_main.py` 的 `__select_difficulty()`。

难度影响：

- 英雄生命：`DIFFICULTY_HERO_HP`
- Boss 普通弹幕频率：`BOSS_INTERVAL_SCALE`
- Boss 全屏弹幕额外频率：`BOSS_FULLSCREEN_INTERVAL_SCALE`

普通弹幕间隔计算：

```python
实际间隔 = 基础间隔 * BOSS_INTERVAL_SCALE[difficulty]
```

全屏弹幕间隔计算：

```python
实际间隔 = 基础间隔 * BOSS_INTERVAL_SCALE[difficulty] * BOSS_FULLSCREEN_INTERVAL_SCALE[difficulty]
```

数值越大，弹幕越稀疏；数值越小，弹幕越密。

## UI 界面

### 难度选择界面

位置：`PlaneGame.__select_difficulty()`

包含：

- 难度按钮。
- 操作说明按钮。
- 操作说明弹窗。

### 暂停界面

位置：`PlaneGame.__show_paused()`

按 `P` 暂停/继续。

### 通关/失败界面

位置：

- `CanvasOver`：失败界面。
- `CanvasClear`：通关界面。

两者复用重新开始和退出按钮。

## 添加新弹幕的推荐方式

1. 在 `danmaku_patterns.py` 的 `DanmakuPattern` 中新增静态方法。
2. 该方法返回一个子弹列表。
3. 在 `Boss.fire_danmaku()` 的某个阶段中调用该方法。
4. 如果属于全屏压制弹幕，建议使用：

```python
self.__on_fullscreen_pattern(...)
self.__fullscreen_group_active(...)
```

5. 如果是特殊弹幕，例如激光、自机狙、阶段演出，可直接使用 `__on_pattern(...)` 保持独立节奏。
